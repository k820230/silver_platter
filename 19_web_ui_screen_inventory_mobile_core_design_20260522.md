## 1. 목적

이 문서는 퀀트 기반 주식 자동매매 프로그램의 Web UI 화면 목록, 주요 화면별 정보 구조, 모바일 브라우저 핵심 화면을 정의한다.

초기 클라이언트는 데스크톱 Web UI를 중심으로 만들고, 모바일은 브라우저 기반 핵심 조회와 알림 확인을 우선한다. 외부 모바일 접속 방식은 추후 결정한다.

## 2. UI 설계 원칙

1. 운영 도구처럼 조용하고 밀도 있게 구성한다.  
   투자, 주문, 리스크, 데이터 상태를 빠르게 스캔할 수 있어야 한다.

2. 현재 모드를 항상 명확히 표시한다.  
   `live`, `paper`, `simulation` 상태가 화면 전체에서 혼동되지 않아야 한다.

3. 주문과 리스크는 같은 흐름에서 본다.  
   주문창은 비용, 세금, 변동성, 위험도, 이벤트 경고를 함께 표시한다.

4. 실시간 알림은 작업을 방해하지 않되 놓치면 안 된다.  
   국제 정세 급변 이벤트는 상단 배너, toast, 알림 패널에 동시에 반영한다.

5. 예측 정보는 신뢰도와 버전을 함께 표시한다.  
   ML 예측, 변동성 지수, 공시 영향 예측은 생성 시각과 모델/산식 버전을 노출한다.

## 3. 전역 레이아웃

### 3.1 데스크톱

```text
Top Bar
  - 현재 모드, 연결 상태, 시장 상태, 긴급 알림

Side Navigation
  - 대시보드
  - 포트폴리오
  - 종목/그룹
  - 주문/체결
  - ML 분석
  - 백테스트/시뮬레이션
  - 공시/헤드라인
  - 리스크
  - 세금
  - 운영

Main Content
  - 화면별 작업 영역

Right Panel
  - 알림, 최근 주문, 데이터 품질, watchlist
```

### 3.2 모바일

모바일은 핵심 조회와 긴급 조치 중심이다.

```text
Top Status
  - 모드, 긴급 알림, 연결 상태

Bottom Navigation
  - 요약
  - 보유
  - 알림
  - 주문
  - 더보기
```

## 5. 홈 대시보드

필수 위젯:

- 총 자산과 당일/누적 손익
- 국내/미국 시장별 평가금액
- KRW/USD 통화 노출
- 위험도 상위 종목
- 사업 그룹별 노출과 한도 근접 상태
- 최근 주문/체결
- 주요 공시와 headline
- 국제 정세 급변 이벤트
- 데이터 수집/브로커 연결 상태
- 마지막 DB 백업 상태

## 6. 종목 상세 화면

탭 구성:

| 탭 | 내용 |
| --- | --- |
| 요약 | 가격, 보유, 손익, 주요 이벤트 |
| 매매 기록 | 매수/매도 체결, FIFO lot, 실현손익 |
| 예측 | ML 가격/거래량/변동성/위험도 예측 |
| 리스크 | 변동성 지수, 위험도 지수, 이벤트 |
| 공시 | 과거 공시와 가격 반응 |
| 헤드라인 | headline과 metadata |

종목별 매도 내역은 어떤 매수 lot과 연결되었는지, lot별 수익금액, 수익률, 보유 기간을 보여야 한다.

## 7. 종목 그룹 화면

### 7.1 그룹 리스트

표시 항목:

- 그룹명
- 표준 산업분류
- 내부 사업 유사도 그룹
- 수동 보정 여부
- 국내/미국 구성 종목 수
- 보유 비중
- 그룹 위험도
- 그룹 변동성
- 최근 headline count
- 한도 근접 상태

### 7.2 그룹 상세

표시 항목:

- 구성 종목
- peer 비교
- 그룹 손익
- 그룹 리스크 점수
- 그룹별 헤드라인
- 공시/이벤트 영향
- 한도 위반 또는 근접 경고

### 7.3 그룹 변동성 비교

사용자는 여러 그룹을 선택해 기준일 대비 변동성 변동 %를 한 그래프에서 비교한다. 기본 가중 방식은 시가총액, 구성 기준은 현재 구성이다.

## 8. 주문창

주문창 필수 영역:

| 영역 | 내용 |
| --- | --- |
| 주문 입력 | 종목, 방향, 수량, 가격, 주문유형 |
| 금액 한도 | 100,000원~1,000,000,000원 상태 |
| 비용 | 수수료, 세금, 환율, 슬리피지 |
| 예측 범위 | 1일/1주/1개월/3개월 가격 범위 |
| 리스크 | 변동성 지수, 위험도 지수, 한도 체크 |
| 이벤트 | 공시, headline, 국제 정세 경고 |
| 매도 preview | FIFO lot, 예상 실현손익, 세금 |
| 전송 제어 | live/paper/simulation 구분, 확인 |

저유동성 종목은 평균 거래대금 기준과 3배 슬리피지 적용 사실을 표시한다.

## 9. ML 분석 화면

필수 탭:

- 가격 예측
- 거래량 예측
- 변동성 예측
- 위험도 예측
- 모델 버전과 성능
- 데이터 품질

표시 기준:

- 예측값과 신뢰구간
- horizon: 1일, 1주, 1개월, 3개월
- 주요 feature
- 최근 예측 오차
- 모델 version
- fine-tuning 상태

## 10. 알림 화면

알림 종류:

- 국제 정세 급변 이벤트
- 공시 영향 예측
- headline risk signal
- 리스크 한도 초과
- 주문 실패
- 데이터 수집 실패
- DB 백업 실패

사용자 조작:

- 확인 처리
- mute
- 심각도 필터
- 관련 종목/그룹 이동
- 수동 검토 등록

모든 조작은 감사 로그에 남긴다.

## 11. 실시간 테스트 모드 화면

필수 표시:

- 현재 simulation session
- 가상 자산과 현금
- 가상 주문/체결
- FIFO 기준 실현/미실현손익
- 리스크 지표
- 실제 시장 데이터 기준 평가금액
- replay 데이터 범위
- 체결 지연과 부분체결 정책

simulation 모드는 화면 전체의 상단 상태와 주문창에서 항상 명확히 표시한다.

## 12. 운영 화면

운영 화면 구성:

- provider 연결 상태
- 데이터 품질
- scheduler 상태
- Goldilocks 연결 상태
- Redis 상태
- DuckDB/Parquet export 상태
- 한국투자증권 Open API 상태
- 마지막 DB 백업과 복구 검증
- API key 보안 점검
- 감사 로그 조회

## 13. 모바일 핵심 화면

### 13.1 모바일 요약

필수 정보:

- 총 자산
- 당일 손익
- 위험도 상위 5개
- 긴급 알림
- 최근 주문 상태

### 13.2 모바일 보유

- 보유 종목 목록
- 손익
- 위험도 점수
- 관련 이벤트 badge
- 종목 상세 이동

### 13.3 모바일 알림

- 국제 정세 급변 이벤트
- 주문 실패
- 리스크 한도 초과
- DB 백업 실패
- 확인/mute

### 13.4 모바일 주문

초기 모바일 주문은 안전하게 제한한다.

- 주문 상태 조회
- 주문 취소
- 수동 승인 요청 확인
- 실거래 주문 신규 입력은 외부 모바일 접속 방식 확정 후 활성화

## 14. 접근 권한

| 권한 | 가능 작업 |
| --- | --- |
| viewer | 조회 |
| trader | 주문 후보 생성, paper/simulation 주문 |
| live_trader | 실거래 주문 승인/전송 |
| admin | 설정, API key, backup, 사용자 관리 |
| auditor | 감사 로그 조회 |

실거래 주문 전송과 API key 설정은 고권한 작업으로 분리한다.

## 15. 상태와 색상 기준

| 상태 | 의미 |
| --- | --- |
| normal | 정상 |
| watch | 관찰 |
| caution | 주의 |
| danger | 위험 |
| crisis | 위기 |
| unavailable | 데이터 없음 |
| degraded | 품질 저하 |

색상은 리스크 지수, 알림, 주문창, 대시보드에서 동일하게 사용한다.

## 16. 테스트 요구사항

| 테스트 | 검증 |
| --- | --- |

| 주문창 한도 | 금액 한도와 경고 |

| FIFO 화면 | lot 매칭 표시 |

| 운영 화면 | 백업/데이터/provider 상태 |
| 접근 권한 | 실거래 권한 분리 |
